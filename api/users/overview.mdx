## Introduction

The User Management system provides comprehensive functionality for managing users and their access within the TechCore API. This system includes user authentication, role-based access control, and user profile management.

## User Model

The User model represents individual users in the system with the following structure:

### User Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | integer | Yes | Unique identifier for the user (auto-generated) |
| `code` | string | Yes | User code (max 10 characters, unique) |
| `nombre` | string | Yes | Full name of the user (max 200 characters) |
| `username` | string | Yes | Unique username for login (max 100 characters) |
| `pwd` | string | Yes | Encrypted password |
| `phone` | string | No | Phone number (max 15 characters) |
| `email` | string | No | Email address (max 200 characters) |
| `idrol` | integer | Yes | Foreign key reference to the user's role |
| `created_date` | datetime | Auto | Timestamp of user creation (defaults to current time) |

### Key Constraints

- **Unique Code**: Each user must have a unique `code` value
- **Unique Username**: The `username` field must be unique across all users
- **Role Assignment**: Every user must be assigned to a valid role via `idrol`
- **Indexed Fields**: The system maintains indexes on `code`, `email`, `idrol`, and `username` for optimized queries

## User Relationships

Users in the system have the following relationships:

- **Role**: Each user belongs to one role (many-to-one relationship with the Rol table)
- **Purchases (Compras)**: Users can create multiple purchase orders
- **Sales (Ventas)**: Users can process multiple sales transactions

## Common Use Cases

### User Registration

When registering new users:
1. Generate a unique user code
2. Validate username uniqueness
3. Hash the password securely
4. Assign an appropriate role
5. Optionally collect phone and email information

### User Authentication

Authentication typically involves:
1. Looking up user by `username`
2. Verifying the password hash
3. Checking the user's role permissions
4. Generating authentication tokens

### User Retrieval

Users can be retrieved by:
- User ID
- Username (unique)
- User code (unique)
- Email address
- Role assignment

## Database Schema

The `users` table is configured with:
- Primary key on `id`
- Unique index on `code`
- Unique index on `username`
- Index on `email` for efficient lookups
- Index on `idrol` for role-based queries
- Foreign key constraint to `rol` table
- Default value for `created_date` (current timestamp)
