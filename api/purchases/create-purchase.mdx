## Endpoint

```
POST /api/purchases
```

Creates a new purchase order in the system. A purchase order consists of header information (supplier, user, dates, totals) and one or more detail lines specifying the products and quantities being purchased.

## Request Body

<ParamField body="norden" type="string" required>
  Unique purchase order number. Maximum 50 characters.
</ParamField>

<ParamField body="ordenN" type="integer" required>
  Sequential order number for internal tracking.
</ParamField>

<ParamField body="codprov" type="string" required>
  Supplier code. Must reference an existing supplier in the system. Maximum 50 characters.
</ParamField>

<ParamField body="codusu" type="integer" required>
  User ID of the employee creating the purchase order. Must reference an existing user.
</ParamField>

<ParamField body="fecha" type="datetime" default="current date/time">
  Purchase order date. Defaults to the current date and time if not provided.
</ParamField>

<ParamField body="subtotal" type="decimal" required>
  Subtotal amount before tax. Precision: 18 digits, 2 decimal places.
</ParamField>

<ParamField body="iva" type="decimal" required>
  Tax amount (IVA). Precision: 18 digits, 2 decimal places.
</ParamField>

<ParamField body="total" type="decimal" required>
  Total purchase amount (subtotal + iva). Precision: 18 digits, 2 decimal places.
</ParamField>

<ParamField body="estado" type="integer" default={1}>
  Purchase order status. Default is 1 (active).
  - `1` - Active
  - `0` - Inactive/Cancelled
</ParamField>

<ParamField body="comprasDetalles" type="array" required>
  Array of purchase detail line items. Each purchase must have at least one detail line.
  
  <Expandable title="Detail Line Object">
    <ParamField body="codprod" type="string" required>
      Product code. Must reference an existing product. Maximum 50 characters.
    </ParamField>
    
    <ParamField body="cantidad" type="integer" required>
      Quantity of the product being purchased. Must be greater than 0.
    </ParamField>
    
    <ParamField body="precio" type="decimal" required>
      Unit price for the product. Precision: 18 digits, 2 decimal places.
    </ParamField>
    
    <ParamField body="subtotal" type="decimal" required>
      Line subtotal (cantidad × precio). Precision: 18 digits, 2 decimal places.
    </ParamField>
  </Expandable>
</ParamField>

## Validation Rules

- `norden` must be unique across all purchase orders
- `codprov` must reference an existing supplier
- `codusu` must reference an existing user
- `comprasDetalles` array must contain at least one item
- Each `codprod` in detail lines must reference an existing product
- `cantidad` must be greater than 0
- `precio` and `subtotal` must be non-negative
- Line `subtotal` should equal `cantidad × precio`
- Header `subtotal` should equal the sum of all detail line subtotals
- `total` should equal `subtotal + iva`

## Notes

- The purchase order date defaults to the current server time if not provided
- Status defaults to 1 (active) if not provided
- Detail line IDs are auto-generated by the database
- Product stock levels are NOT automatically updated when creating a purchase order
- Foreign key relationships ensure data integrity with suppliers, users, and products
