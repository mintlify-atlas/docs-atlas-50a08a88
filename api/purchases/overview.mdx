## Introduction

The Purchases module allows you to create and manage purchase orders from suppliers. Each purchase order consists of a header (Compra) containing general information about the purchase, and detail lines (ComprasDetalle) specifying the products and quantities ordered.

## Key Features

- Create purchase orders with multiple line items
- Track purchases by supplier and user
- Calculate subtotals, tax (IVA), and totals automatically
- Monitor purchase order status
- Link purchases to suppliers and products

## Data Model

### Purchase Order (Compra)

A purchase order represents a transaction with a supplier and contains:

- **Order identification**: Unique order number (norden) and sequential number (ordenN)
- **Parties involved**: Supplier code (codprov) and user who created the order (codusu)
- **Financial details**: Subtotal, IVA (tax), and total amounts
- **Metadata**: Purchase date and status

### Purchase Details (ComprasDetalle)

Each purchase order can have multiple detail lines, each containing:

- Product code and quantity
- Unit price and line subtotal
- Link back to the parent purchase order

## Status Values

The purchase order status field supports the following values:

- `1` - Active (default)
- `0` - Inactive/Cancelled

## Database Schema

### Compras Table

- Primary Key: `norden` (varchar(50))
- Foreign Keys:
  - `codprov` → proveedores.codprovee
  - `codusu` → users.id
- Indexes: codprov, codusu, estado, fecha

### ComprasDetalle Table

- Primary Key: `id` (int, auto-increment)
- Foreign Keys:
  - `norden` → compras.norden
  - `codprod` → productos.codprod
- Indexes: norden, codprod
