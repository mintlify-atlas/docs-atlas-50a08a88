## Overview

The overdue payments system provides real-time tracking of past-due installments with automatic late fee (mora) calculations. This system helps identify delinquent accounts and manage collection efforts.

## Overdue Payment Views

### VwCuotasVencidas (Overdue Installments)

Database view that identifies overdue installments with calculated late fees.

| Property | Type | Description |
|----------|------|-------------|
| `norden` | `string` | Sale order number (max 50 chars) |
| `cliente` | `string` | Customer name (max 200 chars) |
| `numeroCuota` | `int` | Installment number |
| `fechaVencimiento` | `DateOnly` | Original due date |
| `montoCuota` | `decimal(18,2)` | Installment amount |
| `diasAtraso` | `int?` | Number of days past due |
| `moraCalculada` | `decimal(32,4)?` | Calculated late fee amount |

### VwCuotasPorVencer (Upcoming Installments)

Database view that shows installments approaching their due date.

| Property | Type | Description |
|----------|------|-------------|
| `norden` | `string` | Sale order number (max 50 chars) |
| `cliente` | `string` | Customer name (max 200 chars) |
| `numeroCuota` | `int` | Installment number |
| `fechaVencimiento` | `DateOnly` | Due date |
| `montoCuota` | `decimal(18,2)` | Installment amount |

## Late Fee (Mora) Calculation

### Calculation Method

The `moraCalculada` field in `VwCuotasVencidas` is automatically calculated using the following formula:

```
moraCalculada = montoCuota Ã— tasa_mora_diaria Ã— diasAtraso
```

Where:
- `montoCuota`: Original installment amount
- `tasa_mora_diaria`: Daily late fee rate (typically configured in system settings)
- `diasAtraso`: Number of days past the due date

### Example Calculation

For an installment of $5,000 that is 48 days overdue with a 0.1% daily late fee rate:

```
moraCalculada = 5000 Ã— 0.001 Ã— 48 = $240.00
```

## Business Rules

1. **Overdue Determination**: An installment is considered overdue when `fechaVencimiento < CURRENT_DATE` and `pagada = false`
2. **Days Overdue**: Calculated as `CURRENT_DATE - fechaVencimiento`
3. **Late Fee Application**: Late fees are calculated daily and compound over time
4. **Payment Priority**: Payments should be applied to oldest overdue installments first
5. **Grace Period**: Some systems implement a grace period before late fees apply
6. **Maximum Mora**: Late fees may be capped at a maximum percentage of the original amount

## Collection Workflow

1. **Daily Monitoring**: Run daily queries to identify new overdue installments
2. **Alert Generation**: Send automated alerts to customers with upcoming or overdue payments
3. **Escalation Levels**:
   - 1-30 days: Friendly reminder
   - 31-60 days: First collection notice
   - 61-90 days: Second collection notice
   - 90+ days: Escalate to collections department
4. **Mora Assessment**: Calculate and communicate late fees to customers
5. **Payment Negotiation**: Work with customers to establish payment arrangements
