## Overview

The payments system records customer payments (abonos) applied to credit sales. Each payment is linked to a specific sale order and installment number, automatically updating the sale balance through database triggers.

## Payment Model

### AbonosVenta

Represents a payment made by a customer toward a credit sale.

| Property | Type | Description |
|----------|------|-------------|
| `id` | `int` | Unique identifier for the payment |
| `norden` | `string` | Sale order number (max 50 chars) |
| `fecha` | `DateTime?` | Payment date (default: current timestamp) |
| `monto` | `decimal(18,2)` | Payment amount |
| `numeroCuota` | `int` | Installment number this payment applies to |

### Relationships

- **norden** → References `Venta` (Sales)
- Links to `PlanPago` (Payment Plans) by order number and installment number

## Database Triggers

### TR_ActualizarSaldo

Automatically updates the sale balance (`saldo`) when a payment is recorded. This trigger ensures data consistency and eliminates the need for manual balance calculations.

## Database Indexes

The `abonosVentas` table includes the following indexes for optimized queries:

- **IDX_abonosVentas_norden**: Index on sale order number for fast lookups
- **IDX_abonosVentas_fecha**: Index on payment date for date range queries

## Business Rules

1. **Payment Date**: Defaults to current timestamp if not provided
2. **Amount Validation**: Payment amount must be positive and not exceed remaining balance
3. **Installment Reference**: Must reference a valid installment in the payment plan
4. **Balance Update**: Sale balance is automatically updated via trigger
5. **Foreign Key**: Must reference a valid sale order (`norden`)
6. **Installment Status**: When total payments for an installment equal or exceed the installment amount, the installment should be marked as paid

## Payment Workflow

1. Customer makes a payment toward a specific installment
2. Payment is recorded with the sale order number and installment number
3. Database trigger `TR_ActualizarSaldo` automatically:
   - Calculates total payments for the sale
   - Updates the sale's remaining balance (`saldo`)
4. If installment is fully paid, update the payment plan status
5. Generate payment receipt or confirmation
