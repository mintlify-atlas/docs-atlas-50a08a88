## Overview

Payment plans allow customers to pay for sales in installments over time. Each plan consists of multiple installments (cuotas) with specific due dates and amounts.

## Payment Plan Model

### PlanPago

Represents an individual installment in a payment plan.

| Property | Type | Description |
|----------|------|-------------|
| `id` | `int` | Unique identifier for the installment |
| `norden` | `string` | Sale order number (max 50 chars) |
| `numeroCuota` | `int` | Installment number in the sequence |
| `fechaVencimiento` | `DateOnly` | Due date for this installment |
| `montoCuota` | `decimal(18,2)` | Amount due for this installment |
| `pagada` | `bool?` | Payment status (default: false) |

### Relationships

- **norden** → References `Venta` (Sales)
- Linked to `AbonosVenta` (Payments) by order number and installment number

## Database Indexes

The `planPagos` table includes the following indexes for optimized queries:

- **IDX_planPagos_norden**: Index on `norden` for fast sale-based lookups
- **IDX_planPagos_fechaVencimiento**: Index on due dates for date range queries
- **IDX_planPagos_pagada**: Filtered index on unpaid installments (`pagada = 0`)

## Business Rules

1. **Installment Sequence**: Installments are numbered sequentially (1, 2, 3, etc.)
2. **Payment Status**: Default status is unpaid (`pagada = false`)
3. **Due Dates**: Must be in chronological order for sequential installments
4. **Amount Validation**: Total installment amounts should match the sale total plus interest
5. **Foreign Key**: Must reference a valid sale order (`norden`)
