## Get Sale Line Items

<CodeGroup>
```bash GET /api/sales/{norden}/details
curl -X GET "https://api.techcore.com/api/sales/VTA-2026-001/details" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

### Response

<ResponseField name="data" type="array">
  Array of sale detail line items
  
  <ResponseField name="id" type="integer">
    Line item unique identifier
  </ResponseField>
  
  <ResponseField name="norden" type="string">
    Order number reference
  </ResponseField>
  
  <ResponseField name="codprod" type="string">
    Product code
  </ResponseField>
  
  <ResponseField name="cantidad" type="integer">
    Quantity sold
  </ResponseField>
  
  <ResponseField name="pventa" type="decimal">
    Unit price at time of sale
  </ResponseField>
  
  <ResponseField name="subtotal" type="decimal">
    Line item total (cantidad × pventa)
  </ResponseField>
  
  <ResponseField name="codprodNavigation" type="object">
    Product details (if included)
    
    <ResponseField name="codprod" type="string">
      Product code
    </ResponseField>
    
    <ResponseField name="descripcion" type="string">
      Product description
    </ResponseField>
    
    <ResponseField name="precioVenta" type="decimal">
      Current selling price
    </ResponseField>
  </ResponseField>
</ResponseField>

### Example

```json Response
{
  "data": [
    {
      "id": 1,
      "norden": "VTA-2026-001",
      "codprod": "PROD-001",
      "cantidad": 2,
      "pventa": 400.00,
      "subtotal": 800.00,
      "codprodNavigation": {
        "codprod": "PROD-001",
        "descripcion": "Laptop Dell Inspiron 15",
        "precioVenta": 420.00
      }
    },
    {
      "id": 2,
      "norden": "VTA-2026-001",
      "codprod": "PROD-002",
      "cantidad": 1,
      "pventa": 200.00,
      "subtotal": 200.00,
      "codprodNavigation": {
        "codprod": "PROD-002",
        "descripcion": "Mouse Logitech MX Master",
        "precioVenta": 200.00
      }
    }
  ]
}
```

## Record a Payment

<CodeGroup>
```bash POST /api/sales/{norden}/payments
curl -X POST "https://api.techcore.com/api/sales/VTA-2026-002/payments" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "monto": 500.00,
    "numeroCuota": 1,
    "fecha": "2026-03-15T14:20:00Z"
  }'
```
</CodeGroup>

### Request Body

<ParamField body="monto" type="decimal" required>
  Payment amount (18,2 precision)
</ParamField>

<ParamField body="numeroCuota" type="integer" required>
  Installment number being paid
</ParamField>

<ParamField body="fecha" type="datetime">
  Payment date. Defaults to current timestamp if not provided
</ParamField>

### Response

<ResponseField name="id" type="integer">
  Created payment ID
</ResponseField>

<ResponseField name="norden" type="string">
  Order number
</ResponseField>

<ResponseField name="fecha" type="datetime">
  Payment timestamp
</ResponseField>

<ResponseField name="monto" type="decimal">
  Payment amount
</ResponseField>

<ResponseField name="numeroCuota" type="integer">
  Installment number
</ResponseField>

<ResponseField name="saldoActualizado" type="decimal">
  Updated sale balance after payment
</ResponseField>

### Example

```json Response
{
  "id": 1,
  "norden": "VTA-2026-002",
  "fecha": "2026-03-15T14:20:00Z",
  "monto": 500.00,
  "numeroCuota": 1,
  "saldoActualizado": 5250.00
}
```

## Get Payment Schedule

curl -X GET "https://api.techcore.com/api/sales/VTA-2026-002/payment-plan" \
  -H "Authorization: Bearer YOUR_TOKEN"

</CodeGroup>

### Response

<ResponseField name="data" type="array">

## Notes

### Database Triggers

- **TR_DisminuirStock**: Automatically decrements product inventory when sale details are inserted
- **TR_ActualizarSaldo**: Automatically updates sale balance when payments are recorded

### Indexes

The following indexes optimize query performance:

**ventasDetalle**
- `IDX_ventasDetalle_norden`: Fast lookup by order number
- `IDX_ventasDetalle_codprod`: Fast lookup by product

**abonosVentas**
- `IDX_abonosVentas_norden`: Fast lookup by order number
- `IDX_abonosVentas_fecha`: Fast lookup by payment date

**planPagos**
- `IDX_planPagos_norden`: Fast lookup by order number
- `IDX_planPagos_fechaVencimiento`: Fast lookup by due date
- `IDX_planPagos_pagada`: Optimized for unpaid installments

### Business Logic

- Payment amounts automatically reduce the sale's `saldo` field via trigger
- Stock levels are reduced when line items are created and should be restored if sale is voided
- Payment plans should be generated when creating credit sales
- Each payment should reference the appropriate `numeroCuota` from the payment plan
