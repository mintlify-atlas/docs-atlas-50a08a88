---
title: 'Create Sale'
api: 'POST /api/sales'
description: 'Create a new sale transaction with line items'
---

## Request Body

<ParamField body="norden" type="string" required>
  Unique order identifier (max 50 characters)
</ParamField>

<ParamField body="ordenN" type="integer" required>
  Numeric order number for sequential tracking
</ParamField>

<ParamField body="codclien" type="string" required>
  Customer code reference (max 50 characters)
</ParamField>

<ParamField body="codvend" type="integer" required>
  Seller/user ID who processed the sale
</ParamField>

<ParamField body="fecha" type="datetime">
  Sale date and time. Defaults to current timestamp if not provided
</ParamField>

<ParamField body="subtotal" type="decimal" required>
  Subtotal before tax (18,2 precision)
</ParamField>

<ParamField body="iva" type="decimal" required>
  Tax amount (18,2 precision)
</ParamField>

<ParamField body="total" type="decimal" required>
  Total amount including tax (18,2 precision)
</ParamField>

<ParamField body="tipoPago" type="string">
  Payment type: "CONTADO" (cash) or "CREDITO" (installments). Defaults to "CONTADO"
</ParamField>

<ParamField body="meses" type="integer">
  Number of months for installment payments (required if tipoPago is "CREDITO")
</ParamField>

<ParamField body="tasaInteres" type="decimal">
  Interest rate for installments (5,2 precision). Defaults to 0
</ParamField>

<ParamField body="saldo" type="decimal" required>
  Outstanding balance. For cash sales this should be 0, for credit sales equals total initially
</ParamField>

<ParamField body="nula" type="boolean">
  Whether the sale is voided. Defaults to false
</ParamField>

<ParamField body="estado" type="boolean">
  Sale status (active/inactive). Defaults to true
</ParamField>

<ParamField body="ventasDetalles" type="array" required>
  Array of sale line items
  
  <ParamField body="codprod" type="string" required>
    Product code (max 50 characters)
  </ParamField>
  
  <ParamField body="cantidad" type="integer" required>
    Quantity sold
  </ParamField>
  
  <ParamField body="pventa" type="decimal" required>
    Unit price at time of sale (18,2 precision)
  </ParamField>
  
  <ParamField body="subtotal" type="decimal" required>
    Line item subtotal (cantidad × pventa)
  </ParamField>
</ParamField>

## Notes

- The `TR_DisminuirStock` trigger automatically decrements product stock when sale details are created
- For credit sales, you should separately create payment plan entries in the `planPagos` table
- The `norden` field serves as the primary key and must be unique
- Customer (`codclien`) and seller (`codvend`) references must exist in their respective tables
- All product codes in `ventasDetalles` must exist in the products table
