## Get All Customers

```
GET /api/customers
```

Retrieves a list of all customers in the system with optional filtering.

### Query Parameters

<ParamField query="estado" type="boolean">
  Filter customers by active status. Use `true` for active customers, `false` for inactive.
</ParamField>

<ParamField query="nombre" type="string">
  Search customers by name (partial match supported)
</ParamField>

<ParamField query="email" type="string">
  Search customers by email address (partial match supported)
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="integer" default="50">
  Number of records per page (max 100)
</ParamField>

### Response

```json
{
  "data": [
    {
      "codclien": "CUST001",
      "nombre": "Juan Pérez García",
      "telefono": "809-555-1234",
      "email": "juan.perez@example.com",
      "direccion": "Av. Principal #123, Santo Domingo",
      "estado": true,
      "created_date": "2026-02-15T08:30:00Z"
    },
    {
      "codclien": "CUST002",
      "nombre": "María González",
      "telefono": "809-555-5678",
      "email": "maria.gonzalez@example.com",
      "direccion": "Calle Secundaria #456, Santiago",
      "estado": true,
      "created_date": "2026-02-20T14:15:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 50,
    "total": 150,
    "totalPages": 3
  }
}
```

## Get Customer by Code

```
GET /api/customers/{codclien}
```

Retrieves a specific customer by their unique customer code.

### Path Parameters

<ParamField path="codclien" type="string" required>
  The unique customer code
</ParamField>

### Response

<ResponseField name="codclien" type="string">
  The unique customer code
</ResponseField>

<ResponseField name="nombre" type="string">
  Customer's full name
</ResponseField>

<ResponseField name="telefono" type="string">
  Customer's phone number
</ResponseField>

<ResponseField name="email" type="string">
  Customer's email address
</ResponseField>

<ResponseField name="direccion" type="string">
  Customer's physical address
</ResponseField>

<ResponseField name="estado" type="boolean">
  Customer active status
</ResponseField>

<ResponseField name="created_date" type="datetime">
  Timestamp when the customer record was created
</ResponseField>

### Response Example

```json
{
  "codclien": "CUST001",
  "nombre": "Juan Pérez García",
  "telefono": "809-555-1234",
  "email": "juan.perez@example.com",
  "direccion": "Av. Principal #123, Santo Domingo",
  "estado": true,
  "created_date": "2026-02-15T08:30:00Z"
}
```

## Search Customers

```
GET /api/customers/search
```

Performs an advanced search across customer records.

### Query Parameters

<ParamField query="q" type="string" required>
  Search query (searches across name, email, and customer code)
</ParamField>

<ParamField query="estado" type="boolean">
  Filter by customer status
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Maximum number of results to return
</ParamField>

### Response Example

```json
{
  "results": [
    {
      "codclien": "CUST001",
      "nombre": "Juan Pérez García",
      "email": "juan.perez@example.com",
      "estado": true
    },
    {
      "codclien": "CUST042",
      "nombre": "Juan Carlos Méndez",
      "email": "jc.mendez@example.com",
      "estado": true
    }
  ],
  "total": 2
}
```

## Notes

<Info>
  The `nombre`, `email`, and `estado` fields are indexed for optimal query performance. Use these fields in your WHERE clauses for faster searches.
</Info>

<Tip>
  When retrieving customers with sales history, the response includes a `venta` collection containing all sales transactions linked to the customer.
</Tip>
