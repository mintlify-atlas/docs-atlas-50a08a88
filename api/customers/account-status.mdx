## Overview

The Account Status endpoints provide comprehensive views of customer financial information, including outstanding balances, payment history, and credit utilization. These endpoints leverage the `vw_EstadoCuenta` database view for optimized reporting.

## Get Account Status

```
GET /api/customers/{codclien}/account-status
```

Retrieves the complete account status for a specific customer, including all sales orders with outstanding balances.

### Path Parameters

<ParamField path="codclien" type="string" required>
  The unique customer code
</ParamField>

### Response

<ResponseField name="codclien" type="string">
  Customer code
</ResponseField>

<ResponseField name="nombre" type="string">
  Customer name
</ResponseField>

<ResponseField name="orders" type="array">
  Array of sales orders with account status information
  
  <Expandable title="Order object properties">
    <ResponseField name="norden" type="string">
      Sales order number
    </ResponseField>
    
    <ResponseField name="cliente" type="string">
      Customer name (from view)
    </ResponseField>
    
    <ResponseField name="total" type="decimal">
      Total order amount
    </ResponseField>
    
    <ResponseField name="saldo" type="decimal">
      Outstanding balance
    </ResponseField>
    
    <ResponseField name="totalAbonado" type="decimal">
      Total amount paid to date
    </ResponseField>
    
    <ResponseField name="meses" type="integer">
      Number of months in payment plan (if applicable)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="summary" type="object">
  Summary of customer account status
  
  <Expandable title="Summary properties">
    <ResponseField name="totalOrders" type="integer">
      Total number of orders
    </ResponseField>
    
    <ResponseField name="totalRevenue" type="decimal">
      Sum of all order totals
    </ResponseField>
    
    <ResponseField name="totalOutstanding" type="decimal">
      Sum of all outstanding balances
    </ResponseField>
    
    <ResponseField name="totalPaid" type="decimal">
      Sum of all payments made
    </ResponseField>
    
    <ResponseField name="paymentPercentage" type="decimal">
      Percentage of total debt paid
    </ResponseField>
  </Expandable>
</ResponseField>

## Get All Account Statuses

```
GET /api/customers/account-status
```

Retrieves account status information for all customers with outstanding balances.

<ParamField query="minBalance" type="decimal">

</ParamField>

<ParamField query="orderBy" type="string" default="saldo">

</ParamField>

<ParamField query="sortOrder" type="string" default="desc">
  Sort direction. Options: `asc`, `desc`
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="integer" default="50">

</ParamField>

```

```

Provides a high-level overview of a customer's credit utilization and payment behavior.

### Path Parameters

<ParamField path="codclien" type="string" required>

</ParamField>

## Account Status View Schema

The `vw_EstadoCuenta` database view provides the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `norden` | string | Sales order number (max 50 characters) |
| `cliente` | string | Customer name (max 200 characters) |
| `total` | decimal(18,2) | Total order amount |
| `saldo` | decimal(18,2) | Outstanding balance |
| `totalAbonado` | decimal(38,2) | Total amount paid to date |
| `meses` | integer | Number of months in payment plan |

## Notes

<Info>
  The account status views are read-only and reflect real-time data from the sales and payment tables. Changes to sales or payments are immediately reflected in these views.
</Info>

<Note>
  The `totalAbonado` field has a higher precision (38,2) than other decimal fields to accommodate the sum of multiple payment transactions.
</Note>

<Tip>
  Use the `minBalance` parameter to focus on customers with significant outstanding balances when generating collection reports.
</Tip>

<Warning>
  Account status calculations include only non-voided sales (`nula = false`). Voided transactions are excluded from all balance calculations.
</Warning>
