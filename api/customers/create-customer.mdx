## Endpoint

```
POST /api/customers
```

Creates a new customer record in the TechCore system. The customer code (`codclien`) must be unique and will serve as the primary identifier for all future transactions.

## Request Body

<ParamField body="codclien" type="string" required>
  Unique customer code (max 50 characters). This is the primary key and must be unique across all customers.
</ParamField>

<ParamField body="nombre" type="string" required>
  Customer's full name (max 200 characters)
</ParamField>

<ParamField body="telefono" type="string">
  Customer's phone number (max 15 characters)
</ParamField>

<ParamField body="email" type="string">
  Customer's email address (max 200 characters). Indexed for fast lookups.
</ParamField>

<ParamField body="direccion" type="string">
  Customer's physical address (max 300 characters)
</ParamField>

<ParamField body="estado" type="boolean" default="true">
  Customer active status. Set to `false` to create an inactive customer account.
</ParamField>

## Response

<ResponseField name="codclien" type="string">
  The unique customer code
</ResponseField>

<ResponseField name="nombre" type="string">
  Customer's full name
</ResponseField>

<ResponseField name="telefono" type="string">
  Customer's phone number
</ResponseField>

<ResponseField name="email" type="string">
  Customer's email address
</ResponseField>

<ResponseField name="direccion" type="string">
  Customer's physical address
</ResponseField>

<ResponseField name="estado" type="boolean">
  Customer active status
</ResponseField>

<ResponseField name="created_date" type="datetime">
  Timestamp when the customer record was created (auto-generated)
</ResponseField>

## Validation Rules

<AccordionGroup>
  <Accordion title="Customer Code (codclien)">
    - **Required**: Yes
    - **Max Length**: 50 characters
    - **Unique**: Must be unique across all customers
    - **Format**: Alphanumeric, typically uppercase (e.g., "CUST001")
  </Accordion>

  <Accordion title="Name (nombre)">
    - **Required**: Yes
    - **Max Length**: 200 characters
    - **Indexed**: Yes, for efficient name searches
  </Accordion>

  <Accordion title="Phone (telefono)">
    - **Required**: No
    - **Max Length**: 15 characters
    - **Format**: Recommended format includes country/area code
  </Accordion>

  <Accordion title="Email">
    - **Required**: No
    - **Max Length**: 200 characters
    - **Indexed**: Yes, for fast email lookups
    - **Validation**: Should be a valid email format
  </Accordion>

  <Accordion title="Address (direccion)">
    - **Required**: No
    - **Max Length**: 300 characters
  </Accordion>

  <Accordion title="Status (estado)">
    - **Required**: No
    - **Default**: true
    - **Type**: boolean (true = active, false = inactive)
  </Accordion>
</AccordionGroup>

## Notes

<Note>
  The `created_date` field is automatically set to the current timestamp when the customer is created. You cannot set this value manually.
</Note>

<Warning>
  Customer codes cannot be changed after creation. Choose a meaningful and unique code for each customer.
</Warning>

<Tip>
  Use a consistent naming convention for customer codes (e.g., "CUST" prefix followed by sequential numbers) to maintain organization.
</Tip>
