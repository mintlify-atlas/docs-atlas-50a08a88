# Quickstart Guide

Get TechCore API running on your local machine in just a few minutes. This guide will walk you through cloning the repository, setting up the database, and making your first API call.

## Prerequisites

Before you begin, ensure you have the following installed:

- **.NET Core 10 SDK** or later
- **SQL Server** (LocalDB, Express, or full version)
- **Git** for cloning the repository
- **curl** or **Postman** for testing API calls (optional)

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the TechCore API source code to your local machine:

    ```bash
    git clone <repository-url>
    cd source/TechCoreApi
    ```

    <Note>
      The solution includes multiple projects: TechCoreApi (main API), Models (data entities), Concretes (implementations), Interfaces (contracts), and ViewModels (DTOs).
    </Note>
  </Step>

  <Step title="Configure Database Connection">
    Update the connection string in `appsettings.json` to match your SQL Server configuration:

    ```json appsettings.json
    {
      "ConnectionStrings": {
        "DefaultConnection": "Data Source=.\\;Database=TechCore;Trusted_Connection=True;TrustServerCertificate=True;"
      }
    }
    ```

    <Info>
      For SQL Server Express, use `Data Source=.\\SQLEXPRESS`. For a remote server, use `Data Source=your-server-name` and configure authentication accordingly.
    </Info>

    <Warning>
      Make sure SQL Server is running and accepting connections before proceeding to the next step.
    </Warning>
  </Step>

  <Step title="Restore Dependencies">
    Restore all NuGet packages required by the project:

    ```bash
    dotnet restore
    ```

    This will download:
    - Entity Framework Core 10.0.3
    - SQL Server provider
    - Swagger/OpenAPI tools
    - Scalar API documentation
  </Step>

  <Step title="Create and Seed the Database">
    Apply Entity Framework migrations to create the database schema:

    ```bash
    dotnet ef database update
    ```

    This command will create:
    - **Tables**: clientes, productos, ventas, compras, categoria, proveedores, users, rol, planPagos, abonosVentas, ventasDetalle, comprasDetalle
    - **Views**: vw_EstadoCuenta, vw_CuotasVencidas, vw_CuotasPorVencer
    - **Triggers**: TR_ActualizarSaldo (on abonosVentas), TR_DisminuirStock (on ventasDetalle)
    - **Indexes**: Optimized indexes on frequently queried fields

    <Note>
      If migrations don't exist yet in your environment, you may need to generate them first using:
      ```bash
      dotnet ef migrations add InitialCreate
      ```
    </Note>
  </Step>

  <Step title="Run the API">
    Start the development server:

    ```bash
    dotnet run
    ```

    The API will start and automatically redirect the root URL (`http://localhost:5000/`) to the Swagger UI interface.

    <Info>
      By default, the API runs on:
      - HTTP: `http://localhost:5000`
      - HTTPS: `https://localhost:5001`
      
      Check the console output for the exact URLs.
    </Info>
  </Step>

  <Step title="Explore the API">
    Open your browser and navigate to the Swagger UI:

    ```
    http://localhost:5000/swagger
    ```

    You'll see interactive documentation for all available endpoints organized by controller.

    Alternative API documentation is available via Scalar:
    ```
    http://localhost:5000/scalar/v1
    ```
  </Step>
</Steps>

## Implementing Business Endpoints

To add business functionality, create controllers for the database entities. Here's an example for products:

<CodeGroup>
```csharp ProductosController.cs
using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;
using Models.Data;
using Models.Models;

namespace TechCoreApi.Controllers
{
    [ApiController]
    [Route("api/[controller]")]
    public class ProductosController : ControllerBase
    {
        private readonly TechSalesContext _context;

        public ProductosController(TechSalesContext context)
        {
            _context = context;
        }

        [HttpGet]
        public async Task<ActionResult<IEnumerable<Producto>>> GetAll()
        {
            return await _context.Productos
                .Include(p => p.CodCategoriaNavigation)
                .Where(p => p.Estado == true)
                .ToListAsync();
        }

        [HttpGet("{id}")]
        public async Task<ActionResult<Producto>> GetById(string id)
        {
            var producto = await _context.Productos
                .Include(p => p.CodCategoriaNavigation)
                .FirstOrDefaultAsync(p => p.Codprod == id);

            if (producto == null)
                return NotFound();

            return producto;
        }

        [HttpPost]
        public async Task<ActionResult<Producto>> Create(Producto producto)
        {
            _context.Productos.Add(producto);
            await _context.SaveChangesAsync();

            return CreatedAtAction(nameof(GetById), 
                new { id = producto.Codprod }, producto);
        }
    }
}
```
</CodeGroup>

After adding this controller, restart the API and test:

```bash
curl -X GET "http://localhost:5000/api/Productos"
```

## Database Automation Features

TechCore API includes powerful database triggers that automate critical business logic:

<CardGroup cols={2}>
  <Card title="TR_ActualizarSaldo" icon="calculator">
    Automatically updates the outstanding balance (`saldo`) in the `ventas` table whenever a payment is recorded in `abonosVentas`.
  </Card>
  
  <Card title="TR_DisminuirStock" icon="box-open">
    Automatically reduces product stock levels in the `productos` table when sale details are inserted into `ventasDetalle`.
  </Card>
</CardGroup>

## Pre-built Database Views

The API provides several views for common credit management queries:

- **vw_EstadoCuenta**: Complete account statement showing total sales, payments, and balances by customer
- **vw_CuotasVencidas**: Overdue installments with calculated penalties and days overdue
- **vw_CuotasPorVencer**: Upcoming installments due within the next period

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection errors">
    If you see connection errors:
    
    1. Verify SQL Server is running: `sqlcmd -S .\\SQLEXPRESS -Q "SELECT @@VERSION"`
    2. Check your connection string in `appsettings.json`
    3. Ensure Windows Authentication is enabled or use SQL authentication
    4. Verify the database name matches: `TechCore`
  </Accordion>

  <Accordion title="Migration errors">
    If `dotnet ef database update` fails:
    
    1. Ensure Entity Framework tools are installed:
       ```bash
       dotnet tool install --global dotnet-ef
       ```
    2. Check that the Models project is properly referenced
    3. Verify the connection string is correct
    4. Try dropping the database and recreating: `dotnet ef database drop` then `dotnet ef database update`
  </Accordion>

  <Accordion title="Port already in use">
    If the default ports (5000/5001) are in use:
    
    1. Modify `Properties/launchSettings.json` to use different ports
    2. Or set environment variables:
       ```bash
       export ASPNETCORE_URLS="http://localhost:5500"
       dotnet run
       ```
  </Accordion>

  <Accordion title="Swagger UI not loading">
    If Swagger UI doesn't appear:
    
    1. Ensure you're running in Development mode: `export ASPNETCORE_ENVIRONMENT=Development`
    2. Check that the root URL redirects to `/swagger`
    3. Try accessing Swagger directly at `/swagger/index.html`
    4. Verify Swashbuckle packages are installed
  </Accordion>
</AccordionGroup>

<Warning>
  Remember to configure proper authentication and authorization before deploying to production. The current configuration has CORS open to all origins and no authentication middleware enabled.
</Warning>
